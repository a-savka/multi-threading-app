# Отчет о нагрузочном тестировании №1

Дата проведения: 10.11.2025

## 1. Введение

Целью данного тестирования являлась оценка производительности и стабильности веб-приложения при выполнении запросов к эндпоинту `/jobs/status`. Тестирование проводилось для определения базовых метрик пропускной способности (RPS) и времени ответа (latency).

## 2. Параметры тестирования

-   **Инструмент:** Apache Bench (`ab`)
-   **Тестируемый эндпоинт:** `http://localhost:8080/jobs/status`
-   **Уровень параллелизма (Concurrency Level):** 10 одновременных запросов
-   **Общее количество запросов:** 1000

## 3. Результаты тестирования (данные `ab`)

### 3.1. Сводка

| Метрика | Значение |
| :--- | :--- |
| **Пропускная способность (Requests per second)** | **1296.19 RPS** |
| Среднее время на запрос | 7.715 мс |
| Общее время тестирования | 0.771 с |
| Успешных запросов | 1000 |

## 4. Анализ метрик Actuator

Метрики `http.server.requests` были сняты до и после проведения теста для верификации результатов.

### 4.1. Расчеты

-   **Количество запросов (до теста):** 22
-   **Количество запросов (после теста):** 1023
-   **Выполнено запросов во время теста:** 1023 - 22 = **1001**
    *(Примечание: 1000 запросов от `ab` и, вероятно, 1 дополнительный запрос к самому Actuator)*

-   **Суммарное время обработки (до теста):** 0.3405 с
-   **Суммарное время обработки (после теста):** 3.4138 с
-   **Время, затраченное на обработку тестовых запросов:** 3.4138 - 0.3405 = **3.0733 с**

-   **Среднее время обработки одного запроса (по данным Actuator):** 3.0733 с / 1001 ≈ **3.07 мс**

## 5. Выводы

1.  **Пропускная способность:** Приложение продемонстрировало высокую пропускную способность на уровне **~1296 запросов в секунду** для эндпоинта `/jobs/status` при 10 одновременных подключениях.

2.  **Стабильность:** Все 1000 запросов были обработаны **без ошибок**, что свидетельствует о стабильной работе приложения под данной нагрузкой.

3.  **Время ответа:** Время ответа является очень низким. 99% всех запросов были обработаны менее чем за 29 миллисекунд, а медианное время составило всего 6 мс.

4.  **Сравнение `ab` и Actuator:**
    -   Среднее время на запрос по данным `ab` (7.715 мс) включает в себя сетевую задержку и время на обработку на стороне клиента.
    -   Среднее время по данным Actuator (~3.07 мс) отражает "чистое" время, которое сервер потратил на обработку запроса.
    -   Разница между этими двумя значениями (~4.6 мс) является оценкой сетевых и клиентских издержек.

**Итог:** Тестирование показало, что эндпоинт `/jobs/status` работает очень эффективно и стабильно при заданной нагрузке.
